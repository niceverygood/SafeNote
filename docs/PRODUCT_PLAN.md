# 세이프노트 제품 기획 — 두 목적 기준

> **사업자**: 사고 시 입증책임·면책범위 확대(= 다툴 수 있는 증거) · **근로자**: 사고 발생 가능성 감소.
> 모든 기능은 이 두 축 중 하나 이상에 기여해야 한다. "면책 보장" 단정 금지.

## 구현 완료 (운영 배포)
| 기능 | 사업자(입증) | 근로자(예방) |
|---|---|---|
| 면책 자가진단 (9대 의무 rule 매핑·게이지) | 의무 인지·갭 파악 | — |
| 위험성평가 생성기 (rule→RAG→LLM 초안, 매트릭스 확정, 직인, PDF) | 제4조 3호 핵심 증빙 | 위험요인 파악 |
| 근로자 앱: 작업 전·중·후 점검 (고지 확인+체크+서명) | 연속 이행 기록 | 매 작업 위험 인지 |
| 위험 신고 (사진·위치) + 관리자 조치완료 | 발견→개선 루프 증빙 | 위험 제거 |
| **아차사고 신고** (near-miss 유형) | 선제 이행 증거 | 사고 전 단계 제거 |
| **안전 공지·수칙 주지** (관리자 등록 → 근로자 확인 서명, 불변 로그) | 주지 의무 증빙 | 위험 인지율↑ |
| 불변 해시체인 (점검·신고·주지 3체인) + **무결성 검증** | 증거 신뢰성(위변조 탐지) | — |
| **기간별 증빙 리포트 PDF** (점검·신고·조치·주지 + 무결성 검증 결과) | 사고 시 제출 패키지 | — |
| 점검 현황 대시보드 (작업자×전·중·후 매트릭스, 시각·해시) | 이행 관제 기록 | 미점검자 식별 |
| 역할 분리: 총괄(바틀)→관리자 계정 발급→근로자 계정 발급 | 책임 체계 명확화 | — |
| SMS 실시간 알림 (높음 위험 신고 시, env-gated) | 즉시 대응 증빙 | 빠른 조치 |

## 다음 로드맵 (우선순위순)
1. **미점검 리마인더/푸시(FCM)** — 점검 누락 방지(예방) + 상시성 강화(입증)
2. **반기 점검 위저드** — 제4조 반기 1회 점검을 안내·기록화(입증)
3. **교육 콘텐츠 라이브러리** — 업종별 안전교육 자료 배포+이수 서명(주지·교육 증빙)
4. **사고 발생 플레이북** — 사고 직후 해야 할 조치 체크리스트+기록(사고 시 대응 증빙)
5. **보호구 지급 대장** — 지급·수령 서명 기록(입증)
6. **다국어(외국인 근로자)** — 인지율↑(예방)
7. **증빙 외부 앵커링**(체인 해시를 외부 타임스탬프 서비스에 주기 고정) — 증거력 강화
